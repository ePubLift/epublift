# Kobo WebP image plugin

A small Qt image-format plugin (`libqwebp.so`) that teaches **Kobo e-ink
readers** to decode **WebP** images — something the stock firmware cannot do.

Kobo's reading software (Nickel) is built on **Qt 5.2.1**, and WebP support was
only added to Qt's image-format plugins in **Qt 5.3.0** — so Kobo's image stack
simply predates it. The result: WebP images (and WebP covers) show up **blank**
on Kobo, even though the files are perfectly valid (they render fine in Apple
Books and other EPUB 3.3 reading systems). See the full root-cause write-up in
[`docs/device-compatibility.md`](../docs/device-compatibility.md) and
[kobolabs/epub-spec#74](https://github.com/kobolabs/epub-spec/issues/74).

This plugin backports the WebP image-format plugin (from Qt 5.3.0, with its
bundled libwebp) and builds it against Kobo's own Qt 5.2.1, so it loads cleanly
into Nickel and adds WebP decoding.

## What it fixes (and what it doesn't)

Verified first-hand on a **Kobo Forma (N782), firmware 4.38.23697**:

| Path | Before | After plugin |
|------|:------:|:------------:|
| `.kepub.epub` — in-book WebP images | ❌ blank | ✅ render |
| `.kepub.epub` — cover thumbnail | ❌ blank | ✅ renders |
| plain `.epub` — library cover thumbnail | ❌ blank | ❌ **still blank** |

**Important nuance:** the plugin only fixes Nickel's **Qt** image path, which is
what the `.kepub.epub` format uses. Plain `.epub` **library cover thumbnails**
are generated by Kobo's separate, non-Qt (Adobe/RMSDK-style) path, which a Qt
plugin cannot reach — those stay blank regardless. **For WebP to work on Kobo,
use `.kepub.epub`.** (ePubLift can produce one with `--kepub`.)

If you don't want to install anything, the simpler route is to keep covers/images
as JPEG/PNG — ePubLift's `--keep-images` does exactly that and needs no plugin.

## Install (quick)

1. Copy **`KoboRoot.tgz`** into the hidden **`.kobo`** folder at the root of your
   Kobo when connected over USB.
2. Eject safely. The Kobo reboots, shows "Updating…", and installs the plugin to
   `/usr/local/Kobo/imageformats/libqwebp.so`.

Full step-by-step (with testing and uninstall) → [`INSTALL.md`](INSTALL.md).

## Build it yourself

Don't trust a prebuilt binary on your device (you shouldn't!) — the complete,
reproducible from-source recipe is in [`BUILD.md`](BUILD.md). Everything is built
in a Docker container from open-source Qt + libwebp.
